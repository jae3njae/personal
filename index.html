1.2

Overview

what it is

A calm place to think in shapes, not lists.

EnString is a browser-based infinite canvas for organizing ideas. You drop notes, pages, images, tables, checklists, and links onto an endless board, gather them into groups, and connect them with labeled arrows — the strings that give the tool its name.

Most thinking isn't a straight line. Ideas arrive scattered and related; linear note apps flatten that into bullets and lose the relationships. EnString keeps the spatial, relational freedom of a canvas and wraps it in a considered, editorial surface — quiet enough to disappear while you work.

1.3

The building blocks

what it does
Canvas
Infinite board

Pan, zoom, a dotted grid that scales, and a live minimap. Nothing runs out of room.

Content
Seven node types

Rich-text notes, pages you click into, images, tables, checklists, link embeds, and frame groups.

Connections
Strings with meaning

Drag between items to draw an arrow, then label it — “leads to,” “contradicts.”

Control
Made to rearrange

Marquee select, grouping, duplicate, undo/redo, and a layers panel with collapsible groups.

Identity
Three moods

Light, soft, and dark themes share one editorial system, so it matches the room.

Continuity
Workspace, saved

Email or Google sign-in, many boards per account, autosave with an offline-safe queue.

1.4

How it's built

a static front, a managed back

EnString ships as a single self-contained page — no framework, no build step — hosted free as static files on GitHub Pages. Everything that needs a server (accounts, cloud storage) runs on Supabase, so there's no backend to maintain.

Each board is its own database row, saved incrementally. Row-Level Security means an account only ever reaches its own data, every board exports to JSON, and failed saves queue offline and flush on reconnect.
